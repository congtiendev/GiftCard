Trong Magento 2, ACL (Access Control List) là một cơ chế quản lý quyền truy cập dựa trên vai trò người dùng.
sử dụng để định nghĩa và quản lý các quyền truy cập của các vai trò người dùng trong hệ thống Magento Admin,
giúp giới hạn quyền truy cập vào các chức năng và phần của các module cụ thể.

- Cách phân quyền trong Magento :

B1 : Vào trang quản trị chọn mục System > User Roles > Add New Role
    - Role Name : Tên quyền
    - Role Resources : Chọn các mục mà role này có thể truy cập được
    - Nhấn Save Role để lưu lại

B2 : Nếu chưa có user thì tạo user mới bằng cách vào mục System > All Users > Add New User
    - Sau khi tạo user xong thì vào mục User Role để chọn role cho user đó
    Nếu đã có sẵn user chỉ định rồi thì vào mục System > All Users > Edit > User Role để chỉ định role cho user đó

B3 : Tạo file acl.xml trong thư mục etc/adminhtml của module
    - Ví dụ : app/code/Training/HelloWorld/etc/adminhtml/acl.xml
    - Ví dụ : app/code/Training/HelloWorld/etc/acl.xml

    - Cấu trúc file của 1  acl.xml :

    <?xml version="1.0"?>
    <config xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
            xsi:noNamespaceSchemaLocation="urn:magento:framework:Acl/etc/acl.xsd">
        <acl>
            <resources>
            - Trong này sẽ chứa các resource mà module có thể truy cập được theo role

                <resource id="Magento_Backend::admin">  //Đây là tài nguyên cấp cao nhất và thường liên quan đến trang quản lý chung của Magento.

                  <resource id="Mageplaza_MyModule::my_section" title="My Section" sortOrder="10">
                      <resource id="Mageplaza_MyModule::my_subsection" title="My Subsection" sortOrder="20"/>
                  </resource>

                </resource>
            </resources>
        </acl>
    </config>
